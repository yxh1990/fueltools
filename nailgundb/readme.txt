1.说明：
 本工具是用来对fuel的nailgun组件的database进行备份和还原,每次备份都会
 创建一个新的备份文件,为了使用户更好的区分每次备份的区别,在执行备份脚本
 的时候需要传递备份的文件名称.

2.使用场景:
   当用户因为操作失误或者其它任何原因导致UI展示不正常的时候可以快速恢复到最近一次的正常状态

3.建议:
   每次扩容之前把数据先备份一次

4.使用示例:
  1.备份数据库
     python db_backup.py     /root/20170928expansionFront.tar
  2.还原数据库
     python db_recovery.py  /root/20170928expansionFront.tar

5.注意事项
  1.必须使用root用户才能执行备份和删除数据库脚本,否则需要提供密码.
  2.进行数据库备份的时候设置的文件名称必须以.tar结尾,否则可能导致恢复失败.
  3.执行数据库恢复耗用的时间比较长,需要耐心等待.
  4.每次备份设置的文件名称必须要不同
